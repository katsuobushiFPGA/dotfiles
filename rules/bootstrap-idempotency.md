# bootstrap.sh 冪等性ルール

`bootstrap.sh` は **何度実行しても同じ最終状態** になることを保つ。新しいインストール／登録ステップを追加するときは:

- `if ! command_exists <bin>` のように既存チェックでガード
- 副作用のある書き換えは「既に正しいなら何もしない」形で書く
- 移行系の片付け処理（古いバイナリ削除など）も「対象が存在するときだけ」のガードを付ける

## 確認方法

実装した直後に **実機で 2 回連続実行**し、2 回目が no-op（または既に存在する旨の警告のみ）になることを確認する。確認内容:

1. 1 回目: 新規インストール／登録が走る
2. 2 回目: 全ブロックがガードで早期 return／skip する

ブロック単位で確認するのが効率的。例:

```bash
command_exists() { command -v "$1" >/dev/null 2>&1; }
if ! command_exists <tool>; then echo "would install"; else echo "skip"; fi
```

## Why

`bootstrap.sh` は「初回セットアップ」「`mise run init` 経由の再実行」「`mise run update` 経由の継続更新」のいずれでも呼ばれる。冪等でないと、再実行が壊す／重複する／ノイジーな出力を出す。グローバルルールの「動作確認」セクション（`~/.claude/CLAUDE.md`）の延長として、このリポジトリでは特に厳しく守る。
