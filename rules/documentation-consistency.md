# ドキュメント整合性ルール（dotfiles 固有の追補）

グローバルルール（`~/.claude/CLAUDE.md` の「ドキュメント整合性ルール」表）に対する dotfiles リポジトリ固有の追補。

## mise 管理ツールの一覧整合性

`dot_config/mise/config.toml.tmpl` の `[tools]` 配下を編集したら、`README.md` の「その他の CLI ツール（...）は mise が管理する。」という文の括弧内列挙を `[tools]` と **同じコミット**で更新する。

ツールが多くて全部書ききれない場合も、注目すべき新規追加は明示する。「など」で省略していい範囲は、PR レビュー時に既存リスト粒度から判断。

## bootstrap.sh の手書き install ブロックを削除したとき

mise 管理に移行して `bootstrap.sh` から `curl ... | sh` などのブロックを削除した場合、`README.md` の「ツール表」に該当ツールの個別行があれば削除する（mise 管理列に統合される）。同時に「更新セクション」の手動 upgrade コマンドも見直す。
